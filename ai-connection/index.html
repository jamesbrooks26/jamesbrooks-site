The problem

Why most connection requests get ignored

The connection request is the first word in every LinkedIn relationship. Most founders treat it like a formality. They send the default message, or nothing at all, and wait.

The result is a 20 to 30% acceptance rate when it could be 60 to 70%. The problem is not that the person does not want to connect. The problem is that the request gives them no reason to.

A cold ICP sees your request and asks one silent question: why should I let this person into my network? If the request cannot answer that in two sentences, it gets ignored.

The Acceptance Gap is the distance between the number of connection requests you send and the number that actually get accepted.

Most founders accept this gap as normal. It is not. It is a messaging problem, and AI can fix it in under ten minutes.

The shift:

What most founders send

Default LinkedIn message or no message. Generic opener with no reference to shared context. Immediate pitch after acceptance.

What this framework produces

A specific, contextual request that shows you did your homework. A reason to connect that benefits the recipient. Relationship-first sequencing after acceptance.

That single shift changes who says yes.

01

The Context Builder

Generates a personalised connection note based on the ICP's profile and recent activity

What you're doing: Building a connection request that references something specific about the recipient. The goal is to show you have looked at their profile and have a genuine reason to connect, not just a target list.

ROLE: B2B outreach specialist. Your one metric is connection request acceptance rate.

TASK: Write a LinkedIn connection request message using the inputs below. Under 200 characters. No pitch.

INPUTS:
ICP name and role: [job title and company type]
One thing notable about their profile or recent activity: [recent post topic / award / company news / shared connection]
Why connecting benefits them: [what shared context, perspective, or value you bring]

RULES:
- Reference the specific observation in the first sentence
- Never mention your service or offer
- End with a natural open door, not a close
- Under 200 characters
- Output: one connection request message, ready to send
OUTPUT: A personalised connection request under 200 characters, ready to send
02

The Trigger Connector

Writes a connection request tied to a specific buying signal or trigger event

What you're doing: Using the trigger events identified in Resource 01 to write a connection request that arrives at exactly the right moment. When someone posts about a problem your offer solves, the window for a warm connection is open.

ROLE: B2B sales strategist. You specialise in context-triggered outreach.

TASK: Write a LinkedIn connection request using the trigger event below as the opening context. Under 200 characters. No pitch.

INPUTS:
Trigger event: [describe what the ICP posted or did that signals buying intent]
ICP role and company type: [who they are]
What you genuinely found interesting about it: [one honest observation, not flattery]

RULES:
- Reference the trigger directly in the opening
- Sound like a person, not an outreach tool
- Zero service mentions
- Under 200 characters
- Output: one ready-to-send connection request message
OUTPUT: A trigger-referenced connection request that arrives at the right moment
03

The Warm-Up Sequence

Builds a 3-message follow-up sequence for newly accepted connections

What you're doing: The connection request is the opening move. What happens next determines whether the relationship develops or disappears. This prompt builds the first three messages after acceptance, designed to warm the relationship without pitching.

ROLE: B2B relationship strategist. Your focus is building trust before pitching.

TASK: Write a 3-message warm-up sequence for a newly accepted connection using the context below.

INPUTS:
ICP profile: [role / company / challenge they are facing]
Why they accepted: [what common ground or trigger got them to accept]
Your one core value-add: [one thing you can share, observe, or offer that would genuinely be useful to this person]

MESSAGE RULES:
- Message 1 (Day 1): Thank them for connecting. Reference one specific thing from their profile. Under 80 words.
- Message 2 (Day 4–7): Share one piece of value. A resource, observation, or question relevant to their world. No ask. Under 100 words.
- Message 3 (Day 10–14): A soft open door. Something that signals availability without pushing. Under 80 words.

GLOBAL RULES:
- No pitching in any message
- Each message must read like it was written for this one person
- Output: three numbered messages with send-day label
OUTPUT: A 3-message warm-up sequence for newly accepted connections

47 connection requests sent using this framework. 30 accepted. 8 warm conversations opened by message two.

Two calls booked by day 30. Zero cold pitches sent.

What to do right now
  1. Pull your ICP Profile from Resource 01. Paste it into Prompt 1. Generate your three base templates before you send a single request.
  2. Build a list of 20 ICPs to target this week. For each one, note one specific thing about their profile or recent activity.
  3. Use Prompt 2 for any ICP who has posted about a problem in the last 7 days. These are the highest-priority connections right now.
  4. Set up Prompt 3 as a saved template. When someone accepts, send Message 1 the same day.
  5. Track your acceptance rate weekly. If it drops below 50%, rerun Prompt 1 with updated ICP context.
